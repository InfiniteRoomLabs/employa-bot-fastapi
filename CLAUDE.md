# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What This Is

Employa-Bot, forked from the [Full Stack FastAPI Template](https://github.com/fastapi/full-stack-fastapi-template). Two apps in one repo:

- `backend/` -- FastAPI + SQLModel + PostgreSQL, Python 3.14, managed with **uv** (uv workspace; `backend` is the member)
- `frontend/` -- React 19 + TypeScript + Vite + TanStack Router/Query + Tailwind/shadcn, managed with **Bun** (`bun.lock`)

`main.py` at the repo root is untracked PyCharm scaffold junk -- ignore it. The real backend entrypoint is `backend/app/main.py` (`app.main:app`).

## Commands

### Full stack (Docker Compose)

```bash
docker compose watch          # start everything with live sync/reload
docker compose logs backend   # logs for one service
```

URLs: frontend :5173, backend :8000 (docs at /docs), Adminer :8080, Traefik UI :8090, Mailcatcher :1080. Config comes from `.env` at the repo root.

### Backend (from `backend/`)

```bash
uv sync                                  # install deps
fastapi dev app/main.py                  # local dev server (stop the compose backend first)
bash scripts/test.sh                     # coverage run -m pytest tests/ + coverage report/html
uv run pytest tests/api/routes/test_users.py -k test_name   # single test (needs a running DB)
bash scripts/lint.sh                     # mypy (strict) + ty + ruff check + ruff format --check
bash scripts/format.sh                   # ruff autofix + format
```

Tests need the database up. With the stack running, prefer:

```bash
docker compose exec backend bash scripts/tests-start.sh -x   # extra args forwarded to pytest
```

From the repo root, `bash scripts/test.sh` builds and runs the whole stack from scratch, tests, then tears it down.

### Frontend (from `frontend/`)

```bash
bun install
bun run dev            # dev server on :5173
bun run lint           # biome check --write (autofixes)
bun run build          # tsc + vite build
bunx playwright test   # E2E; requires: docker compose up -d --wait backend
```

### Regenerate the API client (after any backend API/schema change)

```bash
bash ./scripts/generate-client.sh   # from repo root; dumps OpenAPI, regenerates frontend/src/client
```

`frontend/src/client/` is generated by @hey-api/openapi-ts -- never hand-edit it.

### Migrations (inside the backend container)

```bash
docker compose exec backend bash
alembic revision --autogenerate -m "Add column X to Y"
alembic upgrade head
```

Every change to `backend/app/models.py` needs a committed alembic revision. Versions live in `backend/app/alembic/versions/`.

### Pre-commit

Uses `prek` (pre-commit replacement): `uv run prek run --all-files`. Hooks: ruff, ruff-format, biome, typos, standard checks.

## Architecture

### Backend (`backend/app/`)

Deliberately flat -- the template pattern is single files, not packages:

- `models.py` -- **all** SQLModel models in one file: table models (`User`, `Item`) and their API schemas (`UserCreate`, `UserPublic`, `UserUpdate`, ...). New entities go here.
- `crud.py` -- all DB access functions.
- `api/routes/*.py` -- one router per resource; registered in `api/main.py`. `api/routes/private.py` is only mounted in local environment.
- `api/deps.py` -- DI: `SessionDep` (DB session), `CurrentUser`, `get_current_active_superuser`. Use these, don't create sessions manually.
- `core/config.py` -- pydantic-settings `Settings`, reads the root `.env` (`../.env`). All config/env vars flow through `settings`.
- `core/security.py` -- JWT + password hashing (pwdlib argon2/bcrypt).
- `core/db.py` -- engine + `init_db`.

Auth flow: `/login/access-token` issues JWT -> `deps.get_current_user` resolves it per request. Routes prefixed `/api/v1` (`settings.API_V1_STR`).

Lint gates are strict: mypy `strict = true`, `ty` with error-on-warning, ruff includes `T201` (no `print`).

### Frontend (`frontend/src/`)

- `routes/` -- TanStack Router file-based routing (route tree is generated).
- `client/` -- generated OpenAPI client; all API calls go through it via TanStack Query.
- `components/` -- shadcn/ui-based components (`components/ui/` is the shadcn primitives).
- Biome is the linter/formatter (not eslint/prettier).

### The backend-change loop

Model change -> alembic revision -> update schemas in `models.py` -> route/crud -> `bash scripts/generate-client.sh` -> frontend picks up new types. Skipping the client regen leaves the frontend typed against a stale API.
