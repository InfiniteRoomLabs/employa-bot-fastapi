# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What This Is

Employa-Bot, forked from the [Full Stack FastAPI Template](https://github.com/fastapi/full-stack-fastapi-template). Two apps in one repo:

- `backend/` -- FastAPI + SQLModel + PostgreSQL, Python 3.14, managed with **uv** (uv workspace; `backend` is the member)
- `frontend/` -- React 19 + TypeScript + Vite + React Router 7 + Tailwind/shadcn, managed with **Bun** (`bun.lock`)

The backend serves two API layers under `/api/v1`: **DB-backed** auth/user routes (`login.py`, `users.py`, `utils.py`) and the **in-memory mock API** implementing the frozen `mvp-api.yaml` contract (89 ops, everything else under `api/routes/`). The mock layer is the placeholder the real persistence/AI implementation replaces. Full mock-API docs (pattern, regen commands, gotchas): `backend/README.md`.

## Commands

### Full stack (Docker Compose)

```bash
docker compose watch          # start everything with live sync/reload
docker compose logs backend   # logs for one service
```

URLs: frontend :5173, backend :8000 (docs at /docs), Adminer :8080, Traefik UI :8090, Mailcatcher :1080. Config comes from `.env` at the repo root.

### Backend (from `backend/`)

```bash
uv sync                                  # install deps
fastapi dev app/main.py                  # local dev server (stop the compose backend first)
uv run pytest -q                         # full suite; POSTGRES_SERVER=localhost with the compose db up
uv run pytest tests/contract -q          # contract tests only -- no DB needed
bash scripts/lint.sh                     # mypy (strict) + ty + ruff check + ruff format --check
bash scripts/format.sh                   # ruff autofix + format
```

DB-backed tests need postgres up. With the stack running, prefer:

```bash
docker compose exec backend bash scripts/tests-start.sh -x   # extra args forwarded to pytest
```

From the repo root, `bash scripts/test.sh` builds and runs the whole stack from scratch, tests, then tears it down.

Heads-up: the test suite's session fixture deletes **all users** in the dev DB at teardown; `docker compose up -d prestart` re-seeds `FIRST_SUPERUSER`.

### Frontend (from `frontend/`)

```bash
bun install
bun run dev            # dev server on :5173 (stop the compose frontend first)
bun run lint           # biome check --write (autofixes)
bun run build          # tsc + vite build
bun run test           # vitest unit/component suite
bunx playwright test   # e2e smoke (e2e/smoke.spec.ts); requires: docker compose up -d --wait backend
```

### Regenerate the API client (after any backend API/schema change)

```bash
bash ./scripts/generate-client.sh   # from repo root; dumps OpenAPI, regenerates frontend/src/client
```

`frontend/src/client/` is generated by @hey-api/openapi-ts -- never hand-edit it. The `generate-frontend-sdk` pre-commit hook keeps it synced.

### Regenerate backend wire models (after a contract change)

`mvp-api.yaml` is frozen; if it changes, regenerate `backend/app/schemas.py` with datamodel-codegen (exact command + post-gen fixup in `backend/README.md`) and update the wire mappers in `frontend/src/data/wire.ts`.

### Migrations (inside the backend container)

```bash
docker compose exec backend bash
alembic revision --autogenerate -m "Add column X to Y"
alembic upgrade head
```

Or outside the container with the compose db up: prefix alembic commands with `POSTGRES_SERVER=localhost` from `backend/`. Every change to `backend/app/models.py` needs a committed alembic revision. Versions live in `backend/app/alembic/versions/`.

### Pre-commit

Uses `prek` (pre-commit replacement): `uv run prek run --all-files`. Hooks: typos, biome, ruff, ruff-format, mypy, ty, generate-frontend-sdk, zizmor, standard checks.

## Architecture

### Backend (`backend/app/`)

Deliberately flat -- single files, not packages:

- `models.py` -- SQLModel **DB** models + their API schemas (`User`, `UserCreate`, `UserPublic`, ...). New DB entities go here.
- `schemas.py` -- **generated** wire models for the mock API (from `mvp-api.yaml`, camelCase verbatim). Regenerate, never hand-edit (one documented post-gen sed).
- `store.py` -- the in-memory mock store: one seeded dict per resource + `reset()`. This is what the real DB layer replaces.
- `crud.py` -- DB access functions (users only, for now).
- `api/routes/*.py` -- one router per resource. `login.py`/`users.py`/`utils.py` are DB-backed; the rest are mock-API routes (pattern exemplar: `searches.py`). All registered in `api/main.py` -- **`resume_lifecycle` must stay included before `resumes`** (path-shadowing comment there).
- `api/errors.py` -- `ApiError` hierarchy + `register_error_handlers(app)`: app-wide handlers that shape errors into the contract's `{kind, path, message}` envelope. Mock routes raise these, never `HTTPException`.
- `api/deps.py` -- DI: `SessionDep` (DB session), `CurrentUser`, `get_current_active_superuser`. Use these, don't create sessions manually.
- `core/config.py` -- pydantic-settings `Settings`, reads the root `.env` (`../.env`). All config/env vars flow through `settings`.
- `core/security.py` -- JWT + password hashing (pwdlib argon2/bcrypt).
- `core/db.py` -- engine + `init_db`.

Auth flow: `/login/access-token` issues JWT -> `deps.get_current_user` resolves it per request. Invalid/expired token -> 403 `"Could not validate credentials"`; missing token -> 401 (the frontend treats both as logout). Routes prefixed `/api/v1` (`settings.API_V1_STR`).

Tests: `tests/contract/` (DB-free, resets the store per test, includes the contract-drift guard asserting all 89 ops are served), `tests/api/` + `tests/crud/` (DB-backed auth/user coverage), `tests/test_operation_manifest.py` (manifest-vs-contract, standalone).

Lint gates are strict: mypy `strict = true`, `ty` with error-on-warning, ruff includes `T201` (no `print`).

### Frontend (`frontend/src/`)

- `routes.ts` -- single-source route table, consumed by `App.tsx` (React Router 7); screens live in `screens/`.
- `data/` -- the data seam: `api.ts` (HTTP adapter over the mock API + auth token handling), `wire.ts` (wire<->app transforms), `types.ts` (frozen app-facing types), `fixtures.ts` (test/Storybook data). Components/hooks never fetch directly -- they go through `hooks/` which wrap `data/api.ts`.
- `client/` -- generated OpenAPI client (from our backend). Currently generated-but-unconsumed; migrating `data/api.ts` onto it is planned follow-up.
- `components/` -- shadcn/ui-based components (`components/ui/` is the shadcn primitives).
- Auth: JWT in `localStorage`; `data/api.ts` attaches `Authorization: Bearer`, and clears + redirects to `/login` on auth failure.
- Biome is the linter/formatter (not eslint/prettier).

### The backend-change loop

DB model change -> alembic revision -> route/crud -> `bash scripts/generate-client.sh`. Contract change -> regenerate `backend/app/schemas.py` + update `src/data/wire.ts` mappers -> mock routes -> client regen. Skipping the client regen leaves `frontend/src/client/` typed against a stale API (the pre-commit hook catches this).
